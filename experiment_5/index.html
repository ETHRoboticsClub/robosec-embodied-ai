Experiment 5

Hijacking GR00T-N1.6 with Adversarial Visual Patches: Model Quirks and Optimized Isolated Patches

Goal

We try to make GR00T-N1.6 grab the bottle while it is told “pick up the orange”, using an adversarial image patch. We map out which interventions actually redirect the grab in closed-loop simulation, and verify each one on rollout video.

Hypothesis

GR00T should be vulnerable to image patches that interact with the object it was told to grab. We expect a patch placed right at an object’s edge, and occluding the requested object, to be the robust ways to redirect the arm — and we ask whether an optimized patch can still redirect the grab from a position isolated from every object, measured against matched controls.

Sub-Experiments

5.2
Four Adversarial-Patch Hijacks of VLA Policies — the campaign’s best results
The strongest results from our patch-attack campaign, each with a matched 0% control: a 60×60 top-left patch makes GR00T grab the bottle under “pick up the orange” (best 0.95 of 2 seeds, mean 0.73); the same patch idea closes a drawer under “pick up the coke can” (best 0.60 redirect of 20 seeds); a physical 12 cm textured box on the table hijacks the grab (0.50, controls 0%); and the attack transfers to π0 (0.45).
Wrapped up
5.1
Hijacking GR00T-N1.6 with Adversarial Visual Patches — What Works, What Doesn’t, and Why
We make GR00T-N1.6 grab the BOTTLE while it is told “pick up the orange” using adversarial image patches. Two robust, video-verified model quirks redirect the grab (a non-occluding patch at the bottle’s top edge, ~65%, and occluding the requested orange, ~35%), plus two optimized patches at an isolated position — a 2D bottom-left patch (~25%) and a physical box on the table (~20% best-case) — that beat a 0% control.
Wrapped up
Overall Conclusion
The redirect on GR00T-N1.6 is real but conditional: a non-occluding patch right at the bottle’s top edge redirects the grab ~65% of the time, and occluding the requested orange makes the arm grab the bottle ~35% of the time. Both are robust, video-verified model quirks — the model is sensitive to a patch at an object’s edge, and to losing sight of the object it was told to grab. Optimized patches placed far from every object — a 2D bottom-left patch (~25%) and a physical box on the table (~20% best-case) — also redirect the grab above a 0% control, though weakly and with high seed-variance.

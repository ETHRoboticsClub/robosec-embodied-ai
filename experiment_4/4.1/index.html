Experiment 4

DAgger-Style Task Steering on GR00T N1.6

Run 81 replaces the old left/right-only story with a six-layout steering matrix that separates scene, object, position, and conjunction effects.

Question

If we learn a fixed boundary delta under the runtime prompt "pick up the orange", does the steered policy follow the target object, the target position, the exact training scene, or some entangled mixture?

Why This Test

The point is not to celebrate one lucky bottle pickup. The point is to identify what variable the boundary intervention actually controls before we invest more time in patching or larger steering runs.

Current Read

The six-layout matrix is more informative than the earlier pilot. The current read is that position steering transfers much more cleanly than object steering, while the object-trained delta mostly looks good on the layouts it saw during training.

Abstract

This page restarts the steering write-up around the current experiment. We keep the DAgger-style training explanation because it still matches the mechanism used here, but we drop the old round-by-round narrative as the main lens. The new core result is a six-layout task-steering matrix on the clean bottle/coke/orange suite. It compares a scene-specific control, an object control, a position control, an object-position conjunction control, and prompt baselines. The main takeaway is not “object steering solved.” The main takeaway is that the present setup exposes a strong and transferable slot / position bias, while clean held-out object transfer is still not established.

Steering Protocol

The model stays frozen. We only learn a fixed delta at the System 2→System 1 boundary. The steering data comes from student-visited states that are relabeled offline with a teacher target. That is the part of the DAgger story that still matters for this page.
student rollout: policy(image, state, "pick up the orange")  →  teacher relabel on the same visited states  →  train one fixed boundary delta  →  eval: policy(image, state, "pick up the orange") + delta
  1. Start from the clean six-layout task-observer collection.
  2. Select training layouts according to the steering hypothesis being tested.
  3. Relabel the chosen orange-prompt states with deterministic teacher targets.
  4. Train one image-token full delta per artifact.
  5. Evaluate each learned delta on all 6 layouts under the unchanged runtime prompt "pick up the orange".

This is why the experiment is interpretable: the prompt at deployment stays fixed, the model weights stay fixed, and only the boundary delta changes from one artifact to another.

Hypothesis Lens

H0. Scene-Specific Steering

Good behavior stays confined to the exact source layout or to a very narrow family.

Signature: strong on train layouts, weak elsewhere.

H1. Object-Level Steering

The delta follows the bottle as bottle moves between left, middle, and right.

Signature: bottle-directed intent survives on held-out layouts.

H2. Position-Level Steering

The delta follows the slot regardless of which object occupies that slot.

Signature: wrong-object failures concentrated on the old slot.

H3. Mixed / Conjunction Code

The learned effect may entangle object and position, or collapse to one factor more than the other.

Signature: conjunction controls look strong only when bottle and slot line up.

For position_right and object_position_bottle_right, intent means consistent attraction to the right slot. That difference is the point of the experiment.

Current Conclusion

The present steering setup is already scientifically useful, but its main result is not the one we originally hoped for. The clean claim from Run 81 is that boundary deltas steer position more robustly than object identity on this suite. Object steering has not yet transferred cleanly across held-out layouts.

Next

Run 81 already answered the main question from this setup. We should not keep extending it just to get more of the same result.

The important result is that the current steering method gives a strong position effect, but not clean object transfer. So the next branch should be a different object test, not another pass on the same matrix.

The useful parts to keep are simple: frozen policy, teacher relabeling, and manual intent review. What needs to change is the training setup if the goal is to isolate object steering.

So the next step is not patching yet. First we need a cleaner experiment where object and position are separated more deliberately.

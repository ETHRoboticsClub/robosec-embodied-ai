GR00T N1.6: LIBERO Is Not
a Reliable Benchmark

Noé Macé
Robotics Security Division, ETHRC
Base Model Fine-tuned Model

Abstract

We evaluate GR00T N1.6 on the Franka Panda embodiment across multiple simulation environments, comparing the zero-shot base model (nvidia/GR00T-N1.6-3B) against 0xAnkitSingh/GR00T-N1.6-LIBERO, fine-tuned on the LIBERO benchmark. Current evaluations are run in LIBERO sim; additional simulators may be added.

Observation — failures are timing issues, not task failures. Inspection of failing episodes reveals the model understands the task and executes the correct sequence of motions in every case. Failures occur because the episode times out before the motion completes — the robot is still on track when the episode ends. This suggests the drop from reference scores is an artifact of episode length limits rather than a genuine capability gap.

Cross-Environment: RoboCasa

The LIBERO fine-tuned model is evaluated on RoboCasa tasks using a PandaOmron fixed-base embodiment — the same Franka Panda arm, but in a completely different simulation environment from the LIBERO training distribution.

Task Fine-Tuned + Prompt
GR00T-N1.6-LIBERO
OpenSingleDoor 0%  (0 / 4)
PnPCounterToCab 0%  (0 / 4)

Complete failure on out-of-distribution environments. Despite near-perfect scores on LIBERO, the fine-tuned model fails on every episode in RoboCasa. The robot arm is identical — the only change is the simulation environment. The model has learned LIBERO-specific scene representations that do not transfer to a new simulator or new task layouts.
